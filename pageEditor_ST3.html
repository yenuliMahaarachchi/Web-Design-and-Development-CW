Team page

Technical Description

The HTML structure of the page follows a well-organized layout that includes a header, navigation bar, content section, and footer. The header contains a navigation menu with links to different pages. The team section is structured using with each team member displayed inside a that contains an image and an information box. The footer contains social media icons and additional navigation links. The CSS styling from Global.css provides a consistent theme, including a background image, flexbox-based navigation bar, and footer alignment. The team.css file focuses on the team member section, utilizing flexbox to arrange members horizontally. The hover provide an interactive experience, enlarging images on hover. The .info-box remains hidden until hovered, ensuring a clean and engaging UI. Overall, the functional and visual aspects are well-aligned for a structured and user-friendly experience.

Accessibility

Accessibility in the Team Page is addressed through several key techniques to ensure inclusivity and usability for all users, including those with disabilities. Alt text is provided for all images, allowing screen readers to describe the content to visually impaired users. Keyboard navigation is enabled using tabindex="0", ensuring users can navigate through interactive elements, such as team member images, without requiring a mouse. Semantic HTML elements like header, nav, main, and footer are used to improve screen reader compatibility and provide a meaningful document structure. The color contrast between text and background is carefully chosen to enhance readability, while hover effects that reveal additional information use smooth transitions, avoiding sudden changes that might disorient users. Additionally, form elements, such as the search bar, include proper labels to assist users relying on assistive technologies. These measures collectively improve accessibility, making the page more user-friendly and compliant with web accessibility standards.

Feedback page

Technical Description

The HTML structure of the feedback page is designed to collect user feedback in a structured format. The fieldset elements group related inputs, with legend providing descriptive labels. The form includes text inputs, radio buttons, checkboxes, dropdowns, and a file upload option, ensuring comprehensive data collection. The header contains a navigation bar, and the footer includes social media links and additional page navigation. The CSS styling from Global.css ensures a consistent background, navigation layout, and footer styling. The feedback.css file enhances form usability, applying styles to form fields for better readability. The .fieldset1 and .fieldset2 classes use a background image and border styling to improve visual appeal, while .emoji: hover adds an interactive rating system. The .button styling ensures clear call-to-action visibility. This combination of structured HTML and well-defined CSS results in a user-friendly and visually engaging feedback form.

Content Page

Technical Description

The HTML structure of the content page presents water conservation techniques in a structured manner using section elements, each containing images and informative text. The navigation menu allows quick access to different sections, improving usability. The header contains a title and a brief description, enhancing readability, while the footer includes social media links and page navigation. The CSS styling from Global.css ensures a consistent theme across the website with background images, structured navigation, and footer alignment. Challenges and Lessons Learned

During the implementation of the various web pages, several challenges were encountered, each requiring thoughtful solutions to maintain functionality, design consistency, and accessibility.

  • One of the primary challenges was ensuring consistent navigation across all pages. With multiple sections and interactive elements, maintaining a structured and responsive navigation menu was difficult. This was resolved by using a flexbox-based navigation bar and a shared CSS file to maintain consistency across all pages.
  • Another challenge was styling and aligning content properly. The Team Page required a dynamic hover effect to display additional information for each team member. Initially, the details box was not displaying correctly. This was fixed by using absolute positioning and opacity transitions to ensure smooth visibility changes. Similarly, in the Feedback Page, aligning form elements while keeping them accessible and user-friendly required using fieldset and legend tags for better grouping.

Compliance

The web pages comply with Janet regulations by ensuring accessibility, data security, and usability best practices.Semantic HTML improves structure and screen reader compatibility, while alt text and keyboard navigation enhance accessibility. The Feedback Page includes input validation and user consent, aligning with data protection standards.Security measures such as relative paths and controlled JavaScript usage prevent vulnerabilities. The Content Page follows readability guidelines with structured sections and high-contrast text.Consistent navigation and responsive design ensure usability across devices. While further enhancements like server-side validation and HTTPS enforcement could improve compliance, the current implementation prioritizes security, accessibility, and ethical web practices.
